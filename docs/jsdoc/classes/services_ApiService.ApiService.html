      Delete a stack via the API

      Parameters

      • stackId: string

        The stack identifier to delete in org/name format

      • accessToken: string

        Bearer token for API authentication

      Returns Promise<void>

      Promise that resolves when deletion is complete

      Remarks

      Throws

    fetchPublicStack

    • fetchPublicStack(stackId: string): Promise<RemoteStack>

      Fetch a public stack without authentication

      Parameters

      • stackId: string

        The stack identifier in org/name format (e.g., "anthropic/public-stack")

      Returns Promise<RemoteStack>

      Promise that resolves to the remote stack data with validation

      Remarks

      Attempts to fetch a stack without authentication headers. This works for public stacks but will fail for private stacks. The method does not automatically fall back to authenticated requests - that should be handled by the caller if needed.

      Throws

      When the API request fails, stack is private, or returns invalid data

      Example 

      const apiService = new ApiService();
      try {
        const stack = await apiService.fetchPublicStack('anthropic/public-stack');
        console.log('Public stack:', stack.title);
      } catch (error) {
        console.log('Stack is private or unavailable');
      }

    publishStack

    • publishStack(
          payload: Record<string, unknown>,
          accessToken: string,
          stackId?: string,
      ): Promise<Record<string, unknown>>

      Publish a stack to the API

      Parameters

      • payload: Record<string, unknown>

        The stack data to publish, including metadata and configuration

      • accessToken: string

        Bearer token for API authentication

      • OptionalstackId: string

        Optional stack ID for updates; omit for new stack creation

      Returns Promise<Record<string, unknown>>

      Promise that resolves to the publish response with server-generated metadata

      Remarks

      Creates a new stack or updates an existing one based on whether stackId is provided. For updates, uses PUT method; for new stacks, uses POST method. The payload is validated server-side and the response is validated client-side.

      Throws

      When upload/update fails due to authentication, validation, or server errors

      Example 

      const apiService = new ApiService();

      // Create new stack
      const newStack = await apiService.publishStack({
        name: 'my-stack',
        description: 'My awesome stack',
        mcpServers: [...]
      }, 'token123');

      // Update existing stack
      const updatedStack = await apiService.publishStack({
        description: 'Updated description'
      }, 'token123', 'anthropic/my-stack');

      Rename a stack via the API

      • newTitle: string

        The new display title for the stack

      • accessToken: string

        Bearer token for API authentication

      Returns Promise<Record<string, unknown>>

      Promise that resolves to the rename response with updated metadata

      Remarks

      Updates the display title of an existing stack without changing its identifier (org/name). The title is used for display purposes in the Commands.com interface and CLI output.

      Throws

      When rename fails due to authentication, permissions, or server errors

      Example

      const result = await apiService.renameStack(
        'anthropic/my-stack',
        'My Awesome Stack v2.0',
        'token123'
      );
      console.log('Stack renamed:', result);

    updateConfig

    • updateConfig(config: Partial<ApiConfig>): void
      Internal

      Update the API configuration

      Parameters

      • config: Partial<ApiConfig>

        Partial configuration object to merge with current config

      Returns void

      Remarks

      Merges the provided configuration with the current configuration. This is primarily intended for testing scenarios where you need to override specific configuration values like baseUrl or timeout. In production, configuration is typically loaded from environment variables.

      Example 

      const apiService = new ApiService();

      // Override base URL for testing
      apiService.updateConfig({
        baseUrl: 'http://localhost:3001',
        timeout: 5000
      });
