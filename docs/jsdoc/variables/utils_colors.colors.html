    Pre-configured color palette for consistent CLI styling across the application

    Provides themed color functions for different types of content including stack names, status messages, file paths, and UI elements. Automatically respects NO_COLOR and FORCE_COLOR environment variables for accessibility compliance.

    Type Declaration

    • author: ChalkInstance

      Styles author names with yellow formatting Used for displaying stack author information

    • bullet: ChalkInstance

      Styles bullet points with magenta formatting Used for list markers and visual separators

    • componentCount: ChalkInstance

      Styles component counts with bright green formatting Used for displaying numerical counts of MCP servers, commands, agents, etc.

    • description: (text: string) => string

      Styles description text with default terminal color Preserves readability by using the terminal's default text color

    • error: ChalkInstance

      Styles error messages with red formatting Used for error conditions and failure notifications

    • highlight: ChalkInstance

      Styles highlighted text with bright magenta formatting Used for emphasis and important text that needs attention

    • id: ChalkInstance

      Styles identifiers with gray formatting Used for IDs, keys, and other identifier strings

    • info: ChalkInstance

      Styles informational messages with bright blue formatting Used for general information and status updates

    • meta: ChalkInstance

      Styles metadata text with dimmed formatting Used for secondary information that should be less prominent

    • number: ChalkInstance

      Styles numbers with cyan bold formatting Used for numerical values, indices, and counts in lists

    • path: ChalkInstance

      Styles file paths with gray formatting Used for displaying file system paths in a subdued manner

    • stackName: ChalkInstance

      Styles stack names with bright blue bold formatting Used for displaying stack identifiers and titles prominently

    • success: ChalkInstance

      Styles success messages with green formatting Used for positive status indicators and completion messages

    • url: ChalkInstance

      Styles URLs with cyan underlined formatting Used for displaying clickable or reference URLs

    • warning: ChalkInstance

      Styles warning messages with yellow formatting Used for cautionary messages and non-critical issues
